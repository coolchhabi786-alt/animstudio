# AnimStudio Frontend ŌĆö Guardrails

## Stack & Conventions

- **Framework**: Next.js 14 App Router (`src/app/` directory)
- **Language**: TypeScript strict mode ŌĆö no `any` except NextAuth session augmentation
- **Styling**: Tailwind CSS with shadcn/ui design tokens (CSS custom properties in `globals.css`)
- **Component Library**: shadcn/ui ŌĆö ALL UI primitives must come from `@/components/ui/*`
- **State Management**: Zustand v4 for client stores, TanStack Query v5 for server state
- **Forms**: react-hook-form v7 + zod schemas + `@hookform/resolvers`
- **Icons**: lucide-react ŌĆö do NOT use emoji or inline SVGs for UI icons
- **Toasts**: sonner via `toast()` ŌĆö never use `alert()` or `console.error()` for user feedback
- **Auth**: next-auth v5 beta; dev mode uses Credentials provider auto-login

## File Naming

- **Components**: kebab-case filenames, PascalCase exports: `character-card.tsx` ŌåÆ `export function CharacterCard`
- **Hooks**: `use-<resource>.ts` (e.g., `use-projects.ts`)
- **Pages**: `page.tsx` inside route-group folders
- **UI primitives**: `@/components/ui/<name>.tsx` ŌĆö generated by shadcn/ui CLI or hand-written following shadcn patterns
- **Feature components**: `@/components/<feature>/<component>.tsx`

## Component Rules

1. Every interactive UI element must use a shadcn/ui primitive (`Button`, `Input`, `Textarea`, `Select`, `Dialog`, `Card`, `Sheet`, etc.)
2. Never use raw HTML `<button>`, `<input>`, `<select>`, or `<textarea>` directly ŌĆö wrap them with shadcn/ui components
3. Always provide accessible labels: `aria-label`, `aria-describedby`, `htmlFor` on `<Label>`
4. Use `cn()` from `@/lib/utils` to merge Tailwind classes conditionally
5. All components must be `"use client"` only when they use hooks or browser APIs
6. Prefer composition over configuration ŌĆö keep components small and focused

## API Integration

- All API calls go through `apiFetch()` from `@/lib/api-client`
- Never use raw `fetch()` directly ŌĆö `apiFetch` handles auth headers, error toasting, and base URL
- API routes use versioned paths: `/api/v1/<resource>` for content/billing, `/api/<resource>` for auth/teams/users
- In dev mode: NO Authorization header is sent (DevAuthHandler auto-authenticates)
- In prod mode: Bearer token from NextAuth session is attached

| Jobs          | `/api/v1/jobs/{id}`                         | Versioned                      |

## Dev Auth

- **Dev User ID**: `00000000-0000-0000-0000-000000000001`
- **Dev Team ID**: `C0000001-0000-0000-0000-000000000001`
- **Dev Email**: `dev@animstudio.local`
- Backend `DevAuthHandler` sets these claims automatically when no `Authorization` header is present
- Frontend middleware bypasses auth checks in `NODE_ENV=development`

## Error Handling

- Use `toast.error()` for user-facing errors (handled by `apiFetch` automatically)
- Use `toast.success()` after successful mutations
- Never swallow errors silently ŌĆö at minimum let `apiFetch` toast them
- For forms, show inline validation via react-hook-form `errors` object

## Performance

- Use TanStack Query `staleTime` to avoid unnecessary refetches (30sŌĆō1hr depending on data volatility)
- Use `queryClient.invalidateQueries()` after mutations to refresh related data
- Use `Skeleton` components from shadcn/ui for loading states ŌĆö never show blank screens

## Testing

- Use Jest + React Testing Library
- Test hooks with `@testing-library/react-hooks` or renderHook
- Mock `apiFetch` in tests, never hit real API

## Do NOT

- Add new dependencies without justification
- Use CSS-in-JS or styled-components ŌĆö Tailwind only
- Use `alert()`, `confirm()`, or `prompt()` ŌĆö use shadcn Dialog/AlertDialog instead
- Import from `next/router` ŌĆö use `next/navigation` (App Router)
- Hard-code API base URLs ŌĆö use `NEXT_PUBLIC_API_BASE_URL` env var
- Store secrets or tokens in localStorage ŌĆö Next-Auth handles session management
