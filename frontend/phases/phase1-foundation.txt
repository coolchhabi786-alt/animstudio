Build the complete production-ready foundation of the AnimStudio SaaS platform.
Use a MODULAR MONOLITH architecture: one deployable .NET 8 Container App with 4
internal bounded-context modules. Each module has its own Clean Architecture layers
(Domain / Application / Infrastructure) and EF Core DbContext scoped to its own
SQL schema. Modules communicate ONLY via MediatR Notifications (domain events) --
never via direct type references to another module's internals.

=== CROSS-CUTTING: Outbox Pattern ===
TransactionBehaviour intercepts every write command:
  1. Begin EF Core transaction
  2. Execute handler (aggregates raise domain events via AddDomainEvent)
  3. Serialize each IDomainEvent to OutboxMessage (EventType + Payload JSON)
  4. Save OutboxMessages + business data atomically in ONE transaction
  5. Commit -- full rollback on any failure; event is never half-written

OutboxPublisherJob (Hangfire, every 1 minute, queue: critical):
  SELECT TOP 50 FROM shared.OutboxMessages WHERE Status=Pending ORDER BY OccurredAt ASC
  FOR EACH message:
    Deserialize Payload -> resolve INotificationHandler via EventType
    Publish via mediator.Publish()
    On success: UPDATE Status=Delivered, ProcessedAt=NOW()
    On failure: UPDATE RetryCount++; if RetryCount>=5: Status=Failed, alert AppInsights

=== CROSS-CUTTING: Resilience (Polly v8) ===
IResiliencePipelineProvider<string> pipelines:
  "service-bus": retry(3, exp backoff + jitter) + circuit-breaker(5 failures, 30s)
  "http-ai-api": timeout(60s) + retry(2, exp backoff) + bulkhead(10 concurrent, 20 queue)
  "http-stripe": timeout(15s) + retry(3, linear 1s) + circuit-breaker(10 failures, 60s)
  "redis":       timeout(500ms) + fallback(log warning, return null / execute DB fallback)

=== CROSS-CUTTING: Redis Cache (ICacheService) ===
Cache key patterns and TTLs:
  user:{userId}                TTL 10min  -- invalidated by UserRegistered, ProfileUpdated
  team:{teamId}                TTL 10min  -- invalidated by TeamCreated, MemberJoined/Left
  team:{teamId}:members        TTL 5min   -- invalidated by MemberJoined/Left
  subscription:{teamId}        TTL 5min   -- invalidated by SubscriptionActivated/Updated
  feature:{teamId}:{feature}   TTL 5min   -- invalidated by SubscriptionActivated/Updated
  plans:all                    TTL 1hr    -- invalidated on Plan admin update
  templates:all                TTL 1hr    -- invalidated on Template admin update (Phase 3)
